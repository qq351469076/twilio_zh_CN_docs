用Python打一个外接电话

现在，我们已经安装了Python和Twilio - Python，我们可以使用刚刚购买的Twilio电话号码发出的一个API请求打一个外接电话。创建一个名为make_call.py的新文件。并在此代码示例中键入或粘贴。

from twilio.rest import Client


# 你的账户Sid和来twilio.com/console上的Auth Token
account_sid = '你的账户Sid'
auth_token = '你的auth_token'
client = Client(account_sid, auth_token)

call = client.calls.create(
                        url='http://demo.twilio.com/docs/voice.xml',
                        to='被呼叫手机号',
                        from_='呼叫人手机号'
                    )

print(call.sid)

用Twilio打个电话

这段代码在我们作为参数传递的两个电话号码之间开始一个电话呼叫。“from”号是Twilio手机号，“to”号是我们要呼叫的人。

URL参数指向某些TwiML，当我们的收件人接电话时，Twilio会告诉他接下来要做什么。这个TwiML告诉Twilio用文本来读消息，然后播放MP3。

安装Flask并设置开发环境

为了处理传入的电话，我们需要一个轻量级web应用程序来接收来自Twilio的传入HTTP请求。我们将使用Flask来实现这个快速启动，但是您可以使用您选择的web框架来完成并接收来自应用程序的电话。

有关在Windows上安装Flask的说明，请参阅本指南。

pip install flask

然后，创建并打开名为answer_phone.py的文件。加上这些行:

from flask import Flask
app = Flask(__name__)

@app.route("/")
def hello():
    return "Hello World!"

if __name__ == "__main__":
    app.run()

现在是时候运行它了。在您的终端中，输入:

python answer_phone.py

您应该看到

$ python answer_phone.py
* Running on http://127.0.0.1:5000/

在浏览器中导航到http://localhost:5000。你应该看到一条“Hello World!”的信息。你已经准备好创建你的第一个Twilio可编程语音应用程序!

允许Twilio与您的Flask应用程序交谈

我们将改进我们的小型Flask应用程序来接受来电。但在此之前，我们需要确保Twilio能够与本地开发环境交互。

大多数Twilio服务使用webhook与应用程序通信。例如，当Twilio收到一个电话时，它会访问应用程序中的一个URL，以获取关于如何处理该呼叫的说明。

当您在开发环境中开发Flask应用程序时，您的应用程序只能被计算机上的其他程序访问，Twilio将无法与之对话。我们需要通过使您的应用程序可以通过internet访问来解决这个问题。

虽然有很多方法可以做到这一点，比如将应用程序部署到Heroku或AWS，但是您可能需要一种不太费力的方法来测试Twilio应用程序。为了使您的应用程序在互联网上可用的一种轻量级方法，我们推荐一种名为ngrok的工具。一旦启动，ngrok就在ngrok.io域名上提供一个惟一的URL，将传入的请求转发到本地开发环境。

点击查看图片

如果您还没有使用ngrok，请转到他们的下载页面，为您的操作系统获取适当的二进制文件。下载后，解压缩包。

如果你使用的是Mac或Linux系统，那么一切都准备好了。如果你使用的是Windows系统，请按照我们的指南在Windows系统上安装和配置ngrok。有关ngrok的更多信息，包括一些很棒的技巧和技巧，请查看这篇深入的博客文章。

下载完成后，启动我们之前创建的Hello World应用程序：

python answer_phone.py

您的本地应用程序必须在本地运行才能使ngrok发挥其魔力。

然后打开一个新的终端选项卡或窗口，并使用以下命令启动ngrok：

ngrok http 5000

如果本地服务器在其他端口上运行，请使用正确的端口号替换5000。

您应该看到与此类似的输出：

点击查看图片

从此输出中复制您的公共URL并将其粘贴到您的浏览器中。你应该看到你的Flask应用程序的“Hello World！” 信息。

使用Twilio响应来电

当你的Twilio号码收到一个来电时，它会向你的服务器发送一个HTTP请求，询问下一步该做什么。一旦您收到请求，您就可以告诉Twilio如何响应该调用。

对于这个快速入门，我们会让我们的Flask应用程序回复电话并给打电话的人发短信。打开answer_phone.py再次更新代码，使其看起来像下面的代码示例:

from flask import Flask
from twilio.twiml.voice_response import VoiceResponse

app = Flask(__name__)


@app.route("/answer", methods=['GET', 'POST'])
def answer_call():
    """接听电话时，给对方发一条简短的信息"""
    # 启动我们的TwiML response
    resp = VoiceResponse()

    # 向打电话的人大声朗读中文信息
    resp.say(u"很开心.", voice='alice', lanuage='zh-CN')

    return str(resp)

if __name__ == "__main__":
    app.run(debug=True)

回复来自Twilio的请求，并说明如何处理调用

最终，Twilio helper会帮助您生成一个类似于这样的XML字符串。

<?xml version="1.0" encoding="UTF-8"?>
<Response>
  <Say voice="alice">hello world!</Say>
</Response>

保存文件并重新启动应用程序

python answer_phone.py

现在您应该能够打开一个web浏览器，访问http://localhost:5000/answer。如果您查看页面源代码，您应该会看到以下文本:

<?xml version="1.0" encoding="UTF-8"?>
<Response>
    <Say voice="alice">Thank you for calling! Have a great day.</Say>
</Response>

这个源代码是在Twilio helper库的帮助下由代码生成的Twilio XML。

再次检查ngrok仍然在本地主机上运行，端口与以前相同。现在Twilio将能够找到您的应用程序。在我们准备好调用你的应用程序之前，我们只需要最后一件事:我们需要告诉Twilio在哪里发送它的请求。

配置您的webhook URL

要让Twilio知道在哪里查找，您需要配置Twilio电话号码，以便在收到新消息时调用webhook URL。

  1. 登录Twilio.com，进入控制台的数字页面。
  2. 点击你可用的语音手机号码。
  3. 找到“Voice & Fax”部分。确保“Accept Incoming”选择被设置为“Voice Calls”。您需要的是默认的“Configure With”选项:“Webhooks/TwiML…”。
  4. 在“A Call Comes In”部分，选择“Webhook”并粘贴你想要使用的URL，附加你的“/answer”路径:

点击查看图片

保存您的更改—您准备好了!

测试你应用

只要您的本地主机和ngrok服务器已经启动并运行，我们就可以开始有趣的部分了——测试我们的新Flask应用程序!

用你的手机打个电话到你的Twilio电话号码。您应该在ngrok控制台中看到HTTP请求。您的Flask应用程序将处理传入的请求并使用TwiML进行响应。然后一旦电话连接你会听到你的信息。
